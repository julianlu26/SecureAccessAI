# Release Publish Steps

Use these steps when publishing the final GitHub release for SecureAccessAI.

1. Open the repository release page and select tag `v1.0.0`.
2. Keep the target branch on `main`.
3. Use `v1.0.0` as the release title.
4. Paste the prepared release notes from `Documentation/Release_Notes_v1.0.0.md`.
5. Verify the latest main branch commit and confirm CI status if available.
6. Publish the release and keep the link for the final submission record.

## Quick Check

- Tag exists on the remote repository.
- Release notes mention authentication, RBAC, threat monitoring, audit logging, and documentation.
- The published release is consistent with the final assessment snapshot.
